# 火眼智擎—汽配领域知产保护分析助手

精准靶向汽配领域侵犯知识产权犯罪的智能分析平台。

## 技术栈

- **后端**: FastAPI + Peewee + SQLite
- **前端**: 独立部署（基于后端API）

## 环境要求

- Python 3.8+

## 访问说明

当前后端启用了单账号密码保护（HTTP Basic）。除 `/`、`/health`、`/docs`、`/openapi.json`、`/redoc` 外，其余接口需要携带账号密码后访问。

默认配置来自环境变量或 `.env`。如果不额外配置，程序也能启动，但会使用示例默认值；实际部署时建议复制 `.env.example` 为 `.env` 并修改密码。

示例文件：

- [.env.example](.env.example)

默认配置项：

- `AUTH_ENABLED=true`
- `AUTH_USERNAME=admin`
- `AUTH_PASSWORD=change-this-password`

## 快速启动

在新设备或不同系统上部署时，建议先按下面步骤准备环境：

1. 复制 [.env.example](.env.example) 为 `.env`
2. 按需修改账号密码和数据库路径
3. 创建并激活 Python 虚拟环境
4. 安装依赖后启动 `python main.py`

```bash
# 安装依赖
pip install fastapi uvicorn peewee pandas openpyxl python-docx

# 启动服务
python main.py

# 访问 API 文档
http://localhost:8000/docs
```

## Docker 一键部署（仅后端）

说明：`streamlit_app.py` 仅用于本地测试，不参与生产部署。Docker 仅启动 FastAPI 后端服务。

分离部署前后端时，后端需要加入共享网络 `huoyan_net`，供前端容器反向代理访问。

1. 准备环境变量：复制 [.env.example](.env.example) 为 `.env` 并修改账号密码
2. 首次部署创建共享网络（仅一次）：

```bash
docker network create huoyan_net
```

3. 一键启动后端：

```bash
docker compose up -d --build
```

4. 访问地址：

- API 文档：[http://localhost:8000/docs](http://localhost:8000/docs)
- 健康检查：[http://localhost:8000/health](http://localhost:8000/health)

5. 常用命令：

```bash
# 查看日志
docker compose logs -f backend

# 停止并移除容器
docker compose down
```

## 主要功能

- **数据导入**: Excel/CSV 文件导入资金流水、通讯记录、物流记录
- **案件管理**: 案件的增删改查
- **智能分析**: 主观明知评分、敏感词检测、角色推断
- **关联分析**: 上下游产业链分析、跨案关联、累犯检测
- **数据台账**: 人员和交易台账管理
- **报告生成**: Word/TXT 分析报告导出

## 业务场景示例

### 场景一：导入三类原始材料后自动形成案件画像

办案人员拿到一批资金流水、通讯记录、物流记录 CSV/Excel，只要按模板导入系统，平台会自动完成：

- 清洗字段并校验表头是否符合模板
- 生成交易、通讯、物流三类预览表
- 抽取人员档案和可疑线索
- 汇总案件分析结果并导出报告

适用场景：公安、市场监管、企业法务对一批零散材料进行快速归并和初筛。

### 场景二：从聊天记录中识别“明知”证据

例如通讯内容出现以下表述：

- “这批高仿原厂标一定要贴好”
- “这批货你别声张，不是原厂的”
- “老规矩，拿货价格再低一点”

系统会结合敏感词库和评分规则，输出：

- 命中关键词
- 主观明知评分
- 涉嫌罪名判断
- 证据原文高亮展示

适用场景：需要证明“明知是假冒注册商标商品仍持续交易”的案件。

### 场景三：从资金流水中梳理交易链条和重点人员

当资金流水中出现“同一收款人反复收款”“多个下游账户集中向同一账户付款”“金额明显偏低”等特征时，系统会：

- 汇总交易笔数和金额
- 识别重点收款人和高频交易对象
- 生成人员台账和交易台账
- 辅助判断是否存在中间商、核心销售者或终端买家

适用场景：对隐蔽的批发、分销、回款链条进行穿透式核查。

### 场景四：跨案碰撞与再犯线索筛查

当同一人员或同一收款账户在多个案件中反复出现时，系统会在关联分析中提示跨案关联，帮助办案人员快速识别：

- 复用的上游供货人
- 复用的收款账户
- 复用的发货地址或物流点
- 多案共用的关键节点

适用场景：排查团伙化、链条化、重复作案的案件线索。

### 场景五：面向出案材料的一键报告输出

当案件初筛完成后，系统可以直接生成分析报告，用于：

- 形成案件摘要
- 汇总可疑证据
- 输出人员台账和交易台账
- 作为内部研判材料或汇报材料的基础版本

适用场景：办案初期需要快速形成可阅读、可汇报、可留痕的阶段性材料。

## 端到端处理流程

一个典型案件通常按下面的路径流转：

1. 上传资金流水、通讯记录、物流记录
2. 系统校验模板格式，格式错误时直接提示用户
3. 清洗数据并提取人员、交易、物流关系
4. 依据敏感词、价格、授权信息计算主观明知和角色
5. 生成可疑线索、台账和案件分析结果
6. 导出 Word/TXT 报告，供人工复核和继续研判

## API 接口

| 模块 | 路径 | 说明 |
|------|------|------|
| 数据导入 | `/api/upload/*` | 上传资金流水/通讯/物流 |

## 核心算法

### 主观明知评分

评分公式：
- 直接承认假冒类 × 5（上限10分）
- 回避性话术类 × 4（上限8分）
- 暗示非正品类 × 2（上限4分）
- 其他类别各 ×1
- 价格低于正品50% +3分
- 无授权证明 +3分
- 无正规进货凭证 +2分

评分等级：
- 8-10分：高度可疑
- 5-7分：中度可疑
- 0-4分：低度可疑

### 敏感词库

7类233+敏感关键词，覆盖：
- 直接承认假冒类
- 回避性话术类
- 暗示非正品类
- 销售行为类
- 生产/贴牌类
- 授权状态类
- 品牌词库
